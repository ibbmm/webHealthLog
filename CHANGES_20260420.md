# 健康管理系统修改说明

## 2026年4月20日修改内容

### 1. 添加血糖记录功能

- **数据库结构**：添加了`blood_sugar`表，包含以下字段：
  - id：主键，自增
  - user_id：用户ID，外键关联users表
  - value：血糖值（mmol/L）
  - fasting：是否空腹（1为空腹，0为餐后）
  - measure_time：测量时间
  - remark：备注
  - created_at：创建时间

- **API端点**：
  - POST `/api/blood-sugar`：添加血糖记录
  - GET `/api/blood-sugar`：查询血糖记录
  - DELETE `/api/blood-sugar/<id>`：删除血糖记录

- **前端界面**：
  - 添加了血糖记录表单，支持录入血糖值、选择是否空腹、测量时间和备注
  - 添加了血糖趋势图表，支持按时间范围和时间段筛选
  - 在历史记录页面添加了血糖记录选项

### 2. 添加用户管理系统

- **数据库结构**：添加了`users`表，包含以下字段：
  - id：主键，自增
  - name：用户姓名
  - created_at：创建时间

- **API端点**：
  - GET `/api/users`：获取用户列表
  - POST `/api/users`：添加用户
  - PUT `/api/users/<id>`：更新用户
  - DELETE `/api/users/<id>`：删除用户（如果没有关联的健康记录）

- **前端界面**：
  - 在设置页面添加了用户管理功能，支持添加、编辑和删除用户
  - 在所有健康记录表单中添加了用户选择下拉菜单
  - 在历史记录和趋势图表页面添加了用户选择下拉菜单

### 3. 改进用户体验

- **表单数据保存**：
  - 实现了表单数据的保存和恢复功能，使用localStorage存储和加载表单数据
  - 确保在刷新页面或重新进入应用程序时，表单会显示上一次使用的数据

- **默认归属人**：
  - 实现了默认使用最近使用的归属人功能
  - 确保所有页面（数据录入、历史记录、趋势图表）都能正确加载最近使用的归属人

### 4. 修复数据库表结构

- 为`blood_pressure`表添加了`user_id`列，并设置默认值为1
- 为`uric_acid`表添加了`user_id`列，并设置默认值为1
- 确保所有表都能正确关联到用户

### 5. 技术优化

- **依赖管理**：更新了`requirements.txt`文件，指定了与Python 3.14兼容的Flask版本
- **代码结构**：改进了前端代码的组织结构，添加了模块化的函数
- **错误处理**：增强了后端API的错误处理能力

### 6. 使用说明

1. **访问应用**：打开浏览器，访问 `http://localhost:5001`
2. **登录**：使用默认密码 `123456` 登录
3. **添加用户**：在设置页面添加用户
4. **录入数据**：在数据录入页面选择用户和健康参数，填写数据并提交
5. **查看历史记录**：在历史记录页面查看历史数据
6. **查看趋势图表**：在趋势图表页面查看数据趋势

### 7. 注意事项

- 所有健康记录都必须关联到一个用户
- 删除用户时，该用户的所有健康记录也会被删除
- 表单数据会自动保存，下次打开时会显示上一次输入的数据
- 归属人会默认选择最近使用的用户，方便快速录入数据

## 技术栈

- **后端**：Flask 2.3.3
- **数据库**：SQLite
- **前端**：HTML、CSS、JavaScript、ECharts
- **部署**：Gunicorn（生产环境）或Flask开发服务器（开发环境）
